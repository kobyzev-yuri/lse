# Отчёт по запросу Керима (12.03.2026): closed по импульсу и совпадение цен

## 1. Новая команда: только позиции с импульсом при входе >5%, без стоп-лоссов

**Сделано:** добавлена команда **`/closed_impulse [N]`** в Telegram-боте.

- **Что выводится:** только закрытые позиции **GAME_5M**, у которых:
  - **импульс при принятии решения об открытии > 5%** — на момент BUY сохраняется `momentum_2h_pct` (импульс 2ч) в `context_json`; фильтр и колонка «Импульс % (при входе)» используют именно его, а не доходность к закрытию;
  - **исключены стоп-лоссы** — строки с причиной выхода `STOP_LOSS` не показываются (отчёт выглядит так, будто стоп не был выставлен).
- **Колонка «Импульс % (при входе)»** — в таблице и в HTML (значение на момент входа; для старых сделок без сохранённого импульса в отчёт не попадают).
- **Не закрытые позиции:** внизу блока выводится блок **«Открытые 5m (ещё не закрыты)»** — тикер, цена входа, текущая цена, импульс в %, дата входа, пометка «не закрыта».

Данные те же, что в основном `/closed`: дата/цена открытия из `trade_history`, факт закрытия и показатели выхода. Отличие — только фильтр (импульс >5%, без STOP_LOSS) и доп. блок по открытым 5m.

## 2. Совпадение цены в уведомлении и в /closed

**Сделано:** после закрытия позиции (запись SELL в БД) крон вызывает **`get_last_closed_for_ticker(engine, ticker)`** и берёт из неё **`entry_price`** и **`exit_price`** для текста уведомления в Telegram. Цифры в сообщении «закрыта по TAKE_PROFIT @ 123.45» теперь совпадают с отчётом `/closed`.

**Причина расхождений (почему раньше могли отличаться):**

- В **/closed** цена входа считается как **средневзвешенная** по всем сделкам BUY по этому тикеру до данного SELL (логика в `compute_closed_trade_pnls`).
- В уведомлении раньше использовалась цена из объекта позиции `open_pos.entry_price`. Она бралась из **`get_open_position_any()`** (тоже средневзвешенная) или из **`get_open_position()`**.
- **`get_open_position()`** возвращает **цену только последнего BUY**, а не среднюю по всем входам. При одном BUY по тикеру всё совпадало; при нескольких входах (добавление к позиции) в уведомлении оказывалась цена последней покупки, а в /closed — средняя, отсюда расхождение.

Сейчас уведомление всегда строится по только что записанной в БД сделке (через `get_last_closed_for_ticker`), поэтому источник данных один и тот же, что и в /closed — расхождений быть не должно.

## 3. Внешний сервис

Отдельный сервис для формирования таблицы /closed не понадобился: фильтрация, расчёт импульса и вывод открытых позиций реализованы внутри бота и отчётов.

---

**Итого:**  
- `/closed_impulse` — закрытые 5m с импульсом >5% без стоп-лоссов + открытые 5m (если не закрылись), с колонкой Impulse %.  
- Цена в уведомлении о закрытии приводится к той же, что в /closed (чтение из БД после записи SELL).  
- Причина прежних расхождений — использование цены последнего BUY вместо средневзвешенной при нескольких входах по одному тикеру.
