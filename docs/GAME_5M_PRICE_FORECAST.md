# Прогноз цены на коротком горизонте (5m)

## Назначение

Для интрадей-игры важнее **диапазон вероятных цен**, чем один тейк. Блок `price_forecast_5m` в выходе `get_decision_5m()` даёт **ориентиры** на **30, 60 и 120 минут** (кратно 5m-бару).

## Метод (v1)

- По последним **5m** закрытиям считаются **лог-доходности** (как в проекте для волатильности).
- На коротком окне оцениваются **σ за один 5m-бар** и **средний дрейф μ** (сжатый к нулю, чтобы не раздувать тренд).
- Суммарная лог-доходность за **H** баров моделируется как **N(H·μ, H·σ²)** (i.i.d. приближение).
- Из этого получаются **квантили цены** p10 / p50 / p90 и **P(цена через горизонт > текущий spot)**.

Это **не** опционная оценка рынка и **не** калиброванная под исторические hit-rates; для принятия решений используйте вместе с RSI, новостями и здравым смыслом.

## Поля в API

- `price_forecast_5m` — объект с `horizons[]` (минуты, p10/p50/p90 цена и % к spot, `p_price_gt_spot`, σ и μ).
- `price_forecast_5m_summary` — одна строка для карточек/Telegram.

## Код

- `services/price_forecast_5m.py` — расчёт.
- Подключение: `services/recommend_5m.py` → `get_decision_5m()`.

## Ограничения

- Гэпы, новости, смена режима волатильности **не** моделируются отдельно.
- При мало барах или нулевой σ прогноз может не строиться (`null`).
