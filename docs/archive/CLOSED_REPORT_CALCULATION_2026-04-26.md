# Расчёт колонок в отчёте закрытых позиций (closed / closed_impulse)

Пояснение на примере таблицы с LITE, MU, SNDK.

---

## Пример таблицы

| Instrument | Dir | Open   | Close  | Импульс % (при входе) | Pips   | Qty | Profit   | ... |
|------------|-----|--------|--------|------------------------|--------|-----|----------|-----|
| LITE       | Long| 679.01 | 677.00 | +12.43%                | -2.01  | 14  | -28.14   | ... |
| MU         | Long| 394.16 | 408.47 | +6.29%                 | 14.31  | +357.75  | ... |
| SNDK       | Long| 589.01 | 624.33 | +8.12%                 | 35.32  | +565.12  | ... |
| ...        |     |        |        |                        |        |          |     |
| **Итого: 5 позиций, суммарный P/L: $+903.93** |

---

## Формулы по колонкам

### Open (цена входа)

- **Средневзвешенная цена входа** по всем лотам, относимым к этой закрытой сделке.
- Для одной пары BUY → SELL по одному тикеру: это просто **цена сделки BUY**.
- Считается в `report_generator.compute_closed_trade_pnls`:  
  `avg_entry = position_cost[ticker] / position_qty[ticker]` на момент продажи.

**LITE:** Open = 679.01 — цена, по которой вошли в длинную позицию (BUY).

---

### Close (цена выхода)

- **Цена сделки SELL** (закрытие позиции).
- Берётся из строки `trade_history` с `side = 'SELL'` по этому тикеру и стратегии.

**LITE:** Close = 677.00 — цена, по которой вышли (продали).

---

### Импульс % (при входе)

- **Не** доходность по сделке (не (Close − Open) / Open).
- Это **импульс на момент решения о входе**: `momentum_2h_pct` по 5m-свечам за ~2 часа до BUY.
- Формула при входе:  
  `(цена_сейчас / цена_24_бара_назад − 1) × 100%`.  
  Сохраняется в `context_json` при BUY (или восстанавливается бэкфиллом).

**LITE:** +12.43% — на момент входа 10.03.2026 16:30 цена за 2 часа выросла на 12.43%; решение «вход» принято при таком импульсе. К моменту выхода (22:45) цена уже откатила, закрытие в минусе по цене.

---

### Причина выхода (Reason)

- **TAKE_PROFIT** — достигнут целевой тейк (уровень задаётся от импульса 2ч при входе: напр. при импульсе 6% тейк ≈ 6% выше входа).
- **TIME_EXIT** — выход по времени: конец сессии (последние N минут при профите ≥ 0.3%) или истечение макс. дней в позиции. Цель тейка при этом могла быть 6%, но цена не дошла — закрылись с минимальным профитом (или малым P/L %).
- **SELL** — закрытие по сигналу SELL.
- **STOP_LOSS** — сработал стоп-лосс.

**Почему при TAKE_PROFIT в отчёте Close может быть заметно ниже локального пика?** В игре `GAME_5M` решение о тейке принимается с учётом **high/окна high** (см. `services/game_5m.py::should_close_position` и `services/recommend_5m.py::build_5m_close_context`), но **в `trade_history.price` при `TAKE_PROFIT` крон записывает `exit_bar_close` последнего доступного 5m-бара** (см. `scripts/send_sndk_signal_cron.py`, ветка `exit_type == "TAKE_PROFIT"`). Поэтому в отчёте `Close` — это **цена закрытия бара “решения”**, а не обязательно high бара, где был максимум окна.

Подробный разбор динамического `take_pct`, окна `recent_bars_high_max`, `session_high` и влияния на `missed_upside` в анализаторе: `docs/GAME_5M_CALCULATIONS_AND_REPORTING.md`.

---

### Qty (количество)

- **Количество проданных лотов** по сделке SELL (в штуках акций/контрактов). Берётся из `trade_history.quantity` для строки закрытия. Нужно для проверки: **Profit ≈ qty × (Close − Open) − commission**.

---

### Pips

- Для **акций** (тикер без `=X`, без подстрок USD/EUR в имени):  
  **разница цен в долларах**, округлённая до 2 знаков:  
  `Pips = round(Close − Open, 2)`.
- Для валютных пар показывается в пипсах (×10000 и округление).

**LITE:**  
`Pips = 677.00 − 679.01 = −2.01` — цена выхода на $2.01 ниже входа.

---

### Profit (чистый P/L в $)

- **Чистая прибыль/убыток по сделке в долларах** с учётом комиссии.
- Формула в `report_generator`:
  - `cost_for_sold = avg_entry × qty` — стоимость проданных лотов по средней цене входа.
  - `proceeds = qty × exit_price − commission` — выручка от продажи минус комиссия.
  - **`net_pnl = proceeds − cost_for_sold`**  
    то есть:  
    **Profit = qty × (Close − Open) − commission**.

**LITE:**  
При Open=679.01, Close=677.00, qty например 14:  
- Убыток по цене: 14 × (677.00 − 679.01) = 14 × (−2.01) = −28.14.  
- После вычета комиссии из выручки итог **Profit = −28.14** (комиссия уже учтена в формуле).

---

### Итого (суммарный P/L)

- Сумма колонки **Profit** по всем строкам отчёта:  
  **Итого = Σ net_pnl**.

**Проверка по вашему примеру:**  
−28.14 + 357.75 + 565.12 + 5.55 + 3.66 = **903.94** ≈ **903.93** (с учётом округлений по сделкам).

---

## Кратко по LITE

- **Вход:** 679.01, при импульсе +12.43% за 2ч (сильный рост на момент BUY).
- **Выход:** 677.00 (откат цены).
- **Pips:** −2.01 (падение цены на $2.01).
- **Profit:** −28.14 $ (убыток с учётом размера позиции и комиссии).
- Импульс при входе (+12.43%) показывает **условия на момент решения**, а не результат сделки; результат отражают Open, Close, Pips и Profit.
