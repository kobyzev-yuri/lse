# Идея решения — MOEX AI-OFFICE Hackathon

**Поле заявки: «Идея решения» (вариант A, развёрнутый)**

---

Мы строим автономного торгового агента под тестовый контур MOEX: после старта бот сам получает рыночные данные по API, оценивает риск и выставляет/снимает заявки без ручного вмешательства. В основе — **двухконтурная архитектура**, согласованная с ограничениями реального времени и с требованием открытых моделей.

**Быстрый контур (исполнение):** детерминированная логика на Python — индикаторы и признаки по цене/объёму (и при необходимости лёгкий табличный ML, например градиентный бустинг), жёсткий **риск-менеджмент** (лимиты по бумагам и портфелю, просадка, ликвидность), учёт **лота, шага цены и издержек**, полное **логирование** состояния портфеля и решений для воспроизводимости.

**Медленный контур (открытый GenAI):** модель семейства **Qwen 3.6** с лицензией **Apache 2.0** через предоставленный организаторами доступ (например OpenRouter) **не на каждом тике**, а для **политики и смысла**: пересмотр параметров стратегии, разбор «спорных» режимов рынка, структурированное обоснование решений в логах. Так мы формально используем генеративный ИИ в составе агента, но **не ставим латентность и стабильность исполнения в зависимость от LLM**.

**Данные:** опираемся на API соревнования; при разрешении внешних источников — официальные и биржевые потоки (в т.ч. корпоративные события MOEX), без зависимости от проприетарных моделей.

**Уникальность:** чёткое разделение «мозг/политика» (LLM) и «нервная система» (код + риск), прозрачность для проверяющих, готовность к оценке и по **доходности**, и по **качеству архитектуры**.
