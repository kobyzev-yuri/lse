# Отчёт для руководства: GAME_5M — несколько стратегий выхода и автономная 30m

В отчёте используются **два** файла результатов:

| Файл | Смысл прогона |
|------|----------------|
| `logs/game5m_take_5m_vs_30m.json` | Фактические **BUY** за 7 дней → реплей выхода на **5m** и на **30m** при том же входе + опционально полная 30m-симуляция по тикерам из BUY. |
| `logs/game5m_sim30m_kb.json` | Только **автономная** 30m-стратегия с **KB+VIX** (`--full-30m-sim --sim-30m-only`), без блока `rows`. |

**Полные таблицы по всем сделкам** из JSON (все пары реплей 5m/30m/факт, сводка по `signal_type`, все сделки автономной 30m) генерируются командой:

```bash
python3 scripts/render_game5m_take_json_to_md.py logs/game5m_take_5m_vs_30m.json -o logs/game5m_take_tables.md
python3 scripts/render_game5m_take_json_to_md.py logs/game5m_sim30m_kb.json -o logs/game5m_sim30m_tables.md
```

Подробное описание полей: `docs/GAME5M_TAKE_5M_VS_30M_REPORT.md`.

---

## 1. Четыре режима в одной логике отчёта

| Стратегия / режим | Тот же вход, что в БД (`BUY`)? | Источник | Учёт KB как в `get_decision_5m` |
|--------------------|---------------------------------|----------|----------------------------------|
| **Реплей выхода 5m** | Да | `market_bars_5m` | Нет (только тейк/стоп по правилам на 5m) |
| **Реплей выхода 30m** | Да | `market_bars_30m` | Нет |
| **Фактический SELL** | Да | `trade_history` | Как было в проде |
| **Автономная 30m** | Нет (свои входы по 30m) | `market_bars_30m` | Да (после техники: sentiment + VIX, **без LLM**) |

Премаркет **1m** ни в реплее, ни в офлайн-30m-симе сейчас **не** подмешивается.

## 3. Сводка по файлу сверки (`logs/game5m_take_5m_vs_30m.json`)

*Источник чисел ниже по блоку A/B — прежняя выгрузка `game5m_take_5m_vs_30m.json` (момент JSON: 2026-04-18T17:13:56 UTC). В том прогоне полная 30m-симуляция в JSON шла **без KB**; актуальный прогон с KB — файл из §2.*

### 3.1. Краткие выводы (блок A: те же входы, два реплея)

1. Из **44** фактических **BUY** за 7 дней **20** строк дают **оба** реплея выхода (5m и 30m). Во всех **20** случаях тип выхода в обоих реплеях — **TAKE_PROFIT**.
2. Средняя \(\ln(P_\text{out}/P_\text{in})\): **5m ≈ 0,0458**, **30m ≈ 0,0474**; средняя разница (**5m − 30m**) **≈ −0,0016** (30m чуть выше).
3. По знаку разницы среди 20 пар: **8** раз лучше 30m, **2** — 5m, **10** — фактически ничья (после округления).
4. У **24** строк нет реплея выхода на доступном хвосте баров (часто нет `SELL` в БД в окне).

*Отрицательная **diff** означает, что **30m-реплей** дал **большую** log-доходность на том же входе.*

### 3.3. Полная таблица «одна позиция — три стратегии выхода»

Для **всех 20** пар (и для всех 44 строк с колонками fill, log_ret, факт `SELL`) используйте сгенерированный Markdown:

## 5. Факт vs реплей 5m (где в JSON есть `actual_sell`)

| Показатель | Значение |
|------------|----------|
| Строк с фактическим `SELL` | **24** |
| Пар «реплей 5m + факт» для сравнения log-return | **18** |
| Среднее (**реплей 5m − факт**) | **≈ +0,0152** log-return |

Реплей использует **high бара** для тейка — оценка **не хуже** фактической цены выхода в БД; для внешней отчётности базовой остаётся **факт**.

## 6. Рекомендации

1. Для сравнения **стратегий на одном входе** опираться на **§1b** рендера из `game5m_take_5m_vs_30m.json`; для **автономной 30m+KB** — на **§2** рендера из `game5m_sim30m_kb.json`.
2. Нарастить выборку по **неделям** и при необходимости расширить список тикеров в `--sim-30m-only`, прежде чем делать выводы о частоте входов автономной 30m.
3. В операционных отчётах явно разделять колонки **факт**, **реплей 5m**, **реплей 30m**, **автономная 30m** — это **разные вопросы** к данным.
