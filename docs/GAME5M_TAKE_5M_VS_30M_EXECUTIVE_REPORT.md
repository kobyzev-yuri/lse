# Отчёт для руководства: GAME_5M — тейк 5m vs 30m и автономная 30m

## 0. Где таблица из **20 сделок** со всеми атрибутами

В этом файле в **git** нельзя хранить выгрузку `logs/game5m_take_5m_vs_30m.json` (она снимается с вашей БД). Поэтому **20 строк** с колонками «тикер, номер BUY, дата входа, цена входа, номер SELL, даты/цены тейка 5m и 30m, разница цен» **не вставлены вручную** — они **всегда строятся из JSON** одной командой.

**Зачем JSON, если база на инстансе.** Скрипт бэктеста читает **PostgreSQL** (`trade_history`, при необходимости `market_bars_*`). На ноутбуке без доступа к той же БД **снять** такой JSON обычно **нельзя** — это не недочёт пайплайна, а разделение ролей: **шаг 1** выполняется **там, где живёт БД** (инстанс, `docker compose exec lse`, …). Зато **шаг 2** (`render … --mode chef`) нужен только **файл JSON**: его можно **скопировать** (`scp` с инстанса) и построить таблицу **полностью офлайн** на любой машине — без VPN к прод-БД и без повторного доступа к базе. То есть JSON — это **переносимый снимок** результата для отчёта и для шефа, а не замена БД в репозитории.

**Шаг 1.** Снять JSON на сервере (пример):

```bash
docker compose exec lse python scripts/backtest_game5m_take_5m_vs_30m.py \
  --days 7 --exchange US --full-30m-sim \
  --json-out /app/logs/game5m_take_5m_vs_30m.json
```

**Шаг 2.** Сгенерировать **одну** понятную таблицу на русском (**ровно те 20 сделок**, где есть и реплей 5m, и реплей 30m):

```bash
python3 scripts/render_game5m_take_json_to_md.py logs/game5m_take_5m_vs_30m.json \
  --mode chef -o docs/GAME5M_20_TRADES_5M_30M_TABLE.md
```

Откройте `docs/GAME5M_20_TRADES_5M_30M_TABLE.md` — там полный перечень.

**Альтернатива:** полный технический отчёт (включая ту же таблицу как **§1d**):

```bash
python3 scripts/render_game5m_take_json_to_md.py logs/game5m_take_5m_vs_30m.json -o logs/game5m_take_tables.md
```

### Расшифровка колонок «шеф-таблицы» (§1d / `--mode chef`)

| Колонка в таблице | Что это |
|-------------------|---------|
| **Тикер** | Символ из строки `BUY`. |
| **№ BUY в БД** | `buy_id` — первичный ключ покупки в `trade_history`. |
| **Дата и время входа** | `entry_ts` фактического `BUY` (как в БД). |
| **Цена входа (факт BUY)** | `entry_price` — одна цена на оба реплея; реплей отвечает на вопрос «когда бы вышли с **этой** позиции». |
| **№ SELL в БД** | `id` следующей записи `SELL` после этого `BUY`, **если** она уже есть; к датам тейка в реплее **не обязан** совпадать. |
| **Дата и время тейка (реплей 5m)** | Момент закрытия бара 5m, на котором впервые сработал бы выход (`bar_end_et` в JSON). |
| **Цена тейка (реплей 5m)** | `exit_fill_price` реплея 5m (при `TAKE_PROFIT` — по правилам игры, опора на **high** бара). |
| **Дата и время тейка (реплей 30m)** | То же для сетки **30m**. |
| **Цена тейка (реплей 30m)** | `exit_fill_price` реплея 30m. |
| **Разница цен тейка (5m−30m), USD** | Цена тейка 5m **минус** цена тейка 30m; отрицательное значение → по реплею **дороже** вышли бы на 30m. |
| **Разница log_ret (5m−30m)** | \(\ln P_{5m}/P_\text{вход} - \ln P_{30m}/P_\text{вход}\) из JSON (`log_ret_diff_5m_minus_30m`). |
| **Тип выхода 5m / 30m** | `signal_type` (в выгрузке за неделю по **20** парам это везде **TAKE_PROFIT**). |

Второй файл результатов — **автономная 30m + KB** (другой сценарий, свои входы): `logs/game5m_sim30m_kb.json` и рендер без `--mode chef` (в §2 — сделки автономной стратегии).

Подробное описание полей JSON: `docs/GAME5M_TAKE_5M_VS_30M_REPORT.md`.

## 1. Четыре режима в одной логике отчёта

| Стратегия / режим | Тот же вход, что в БД (`BUY`)? | Источник | Учёт KB как в `get_decision_5m` |
|--------------------|---------------------------------|----------|----------------------------------|
| **Реплей выхода 5m** | Да | `market_bars_5m` | Нет (только тейк/стоп по правилам на 5m) |
| **Реплей выхода 30m** | Да | `market_bars_30m` | Нет |
| **Фактический SELL** | Да | `trade_history` | Как было в проде |
| **Автономная 30m** | Нет (свои входы по 30m) | `market_bars_30m` | Да (после техники: sentiment + VIX, **без LLM**) |

Премаркет **1m** ни в реплее, ни в офлайн-30m-симе сейчас **не** подмешивается.
