## 1. Расшифровка колонок «шеф-таблицы» (§2)

| Колонка | Смысл |
|---------|--------|
| **Тикер** | Символ из строки `BUY`. |
| **№ BUY в БД** | `buy_id` в `trade_history`. |
| **Дата и время входа** | `entry_ts` фактического `BUY`. |
| **Цена входа (факт BUY)** | Одна цена на оба реплея. |
| **№ SELL в БД** | `id` следующего `SELL`, если уже записан; с датами реплея не обязан совпадать. |
| **Дата/время тейка (реплей 5m / 30m)** | Закрытие бара, на котором впервые сработал бы выход. |
| **Цена тейка (реплей …)** | `exit_fill_price` (при `TAKE_PROFIT` — по правилам игры, **high** бара). |
| **Разница цен (5m−30m), USD** | Цена тейка 5m минус 30m. |
| **Разница log_ret (5m−30m)** | Из JSON, поле `log_ret_diff_5m_minus_30m`. |
| **Тип выхода 5m / 30m** | `signal_type` (в этом снимке по 20 парам — **TAKE_PROFIT**). |
