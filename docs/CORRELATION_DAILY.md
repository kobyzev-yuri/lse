# Дневная корреляция тикеров (кластер / game 5m)

## Проблема

Жёсткое требование «в один календарный день есть close **почти у всех** символов» (например `dropna(thresh=n−2)` по строкам) даёт мало общих дней для смеси **акции + индексы + FX + фьючерсы**: календари и обновление БД расходятся, в понедельник утром — ещё хуже. При этом **попарная** корреляция по лог-доходностям между двумя тикерами считается только по дням, где есть **оба** значения — остальные столбцы в эти даты не обязаны быть заполнены.

## Надёжный режим (`for_correlation=True`)

В `ClusterManager.get_price_data_with_fallback`:

1. **БД:** `min_tickers_per_row=2` — оставляем день, если есть данные хотя бы по двум тикерам (максимум пересечения дат).
2. Если строк **&lt; 5** — **yfinance:** для каждого символа дневной `Close`, индекс приводится к naive дате, столбцы объединяются **outer** по датам. **Нет** `dropna(thresh=…)` по строкам.
3. **Лог-доходности:** `log(C_t/C_{t-1})`, строки где все NaN — удаляются (`dropna(how="all")`).
4. **Матрица:** `DataFrame.corr(min_periods=5)` — для каждой пары свой минимум из 5 совместных наблюдений (правило проекта: log-returns).

Предупреждение «меньше 5 строк из БД» понижено до **debug**: при корреляции почти всегда есть переход на yfinance.

## Где включено

- `get_correlation_and_beta_matrix()` — всегда `for_correlation=True`.
- Команда `/corr` в Telegram — тот же флаг.

См. реализацию: `services/cluster_manager.py` (`_fetch_daily_closes_yfinance_outer`, `get_price_data_with_fallback`).

## Почему `/corr` (все тикеры цикла) ок в Telegram, а крон 5m пишет в лог иначе

Это **разные наборы символов**, не «тот же расчёт, что в боте»:

| | Telegram **`/corr`** | Крон **`send_sndk_signal_cron`** (блок корреляции) |
|---|----------------------|---------------------------------------------------|
| **Откуда список** | `get_tickers_for_portfolio_game()` — `TRADING_CYCLE_TICKERS` или MEDIUM+LONG (портфельная игра) | `get_tickers_for_5m_correlation()` — **только игра 5m** + свой контекст (`GAME_5M_CORRELATION_CONTEXT`: MSFT, QQQ, ^VIX, нефть, forex и т.д.) |
| **Смысл** | корреляции по тому же кластеру, что портфель / trading cycle | корреляции для **промпта игры 5m** (быстрые стоки + фон) |

Команда **`/corr5m`** — отдельно: там только `get_tickers_game_5m()`, без портфеля и без расширенного 5m-контекста.

**Почему в боте тише:** нет фонового cron каждые 5 мин; сообщения про «БД мало дней → yfinance» не сыпятся в файл логов.

**Что уже сделано в коде:** для длинного списка (как у крона) загрузка истории выровнена по масштабу с ботом (`max(days+30, 252)`), fallback БД → yfinance в логах — **debug**, чтобы не путать с ошибкой.

Если нужно, чтобы крон считал **ровно тот же** набор, что `/corr`, это отдельное изменение логики (подставить портфельный список вместо `get_tickers_for_5m_correlation()` — но тогда матрица перестанет совпадать с задумкой «игра 5m + макро/сектор»).
