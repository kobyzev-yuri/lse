# Новости в решении игры 5m

## Текущее состояние

Блок **«LLM-новости»** в отчёте 5m (/prompt_entry game5m, /recommend5m, крон) **по умолчанию выключен**.

- Конфиг: `GAME_5M_USE_LLM_NEWS=false` (или не задан). При `true` в решении 5m вызывается `fetch_news_for_ticker(ticker)`.
- Причина отключения: модель отвечает **по своим обучающим данным**, без доступа в интернет в реальном времени. В тексте часто оказываются даты 2023 года и старше — это не актуальные новости, а «память» модели. Показывать их как «новости» вводят в заблуждение.

Актуальные новости по тикерам уже есть в блоке **«Новости из KB»**: их собирает крон (RSS, NewsAPI, Investing.com, Alpha Vantage и т.д.). Решение 5m учитывает их через `kb_news`, `kb_news_impact`, sentiment по KB.

---

## Когда включить блок снова

Включать `GAME_5M_USE_LLM_NEWS=true` имеет смысл только когда источник новостей для LLM будет **действительно актуальным**. Ниже варианты.

---

## Варианты актуальных новостей для 5m

### 1. Только KB (уже есть)

- **Что:** не использовать отдельный «LLM-новости» блок; контекст по новостям — только из `knowledge_base` (get_recent_news по тикеру/датам).
- **Плюсы:** уже реализовано, даты реальные (RSS, NewsAPI, Investing.com).
- **Минусы:** нет суммаризации/инсайта от LLM по этим новостям.
- **Действие:** оставить `GAME_5M_USE_LLM_NEWS=false`. В отчёте остаётся блок «Новости из KB».

### 2. RAG по свежим записям KB

- **Что:** не спрашивать LLM «какие новости по тикеру», а передавать в LLM **уже выбранные** свежие тексты из KB (например за последние 1–3 дня), пусть модель даёт краткий инсайт и sentiment.
- **Плюсы:** даты и факты — из реальных источников; LLM только интерпретирует.
- **Минусы:** нужна доработка: выборка по дате/тикеру, обрезка по длине, промпт «суммаризуй и дай sentiment по этим новостям».
- **Реализация:** в `get_decision_5m` (или отдельный сервис) брать `get_recent_news(ticker, days=2)`, подставлять в промпт, вызывать LLM; результат класть в `llm_news_content` / `llm_sentiment` / `llm_insight`. Тогда можно включить `GAME_5M_USE_LLM_NEWS=true` и переименовать блок, например в «Инсайт LLM по новостям из KB».

### 3. Web Search API (Bing, Serper, Tavily и др.)

- **Что:** перед решением 5m выполнять поиск в интернете по запросу вида «{ticker} stock news last 24 hours», отдавать сниппеты/ссылки в LLM для суммаризации и sentiment.
- **Плюсы:** реально свежие новости, в т.ч. breaking.
- **Минусы:** отдельный API, лимиты, задержка; нужна фильтрация по дате и релевантности.
- **Реализация:** сервис по аналогии с `llm_news_fetcher`, но с шагом поиска (Serper/Bing/Tavily) и промптом «по этим результатам поиска дай 2–5 пунктов и sentiment». Результат — в тот же блок отчёта (тогда логично переименовать в «Новости (web + инсайт LLM)»).

### 4. Гибрид: KB + опционально web

- **Что:** по умолчанию контекст из KB; при наличии ключа Web Search API — дополнять поиском за последние N часов и отдавать в LLM для инсайта.
- **Плюсы:** можно обойтись без платного поиска (только KB) или включить поиск при необходимости.
- **Реализация:** флаг в config (например `GAME_5M_NEWS_USE_WEB_SEARCH=false`), при `true` — вызывать поиск и объединять с выборкой из KB перед вызовом LLM.

---

## Рекомендация

- **Сейчас:** оставить `GAME_5M_USE_LLM_NEWS=false`, опираться на «Новости из KB» и тех. контекст.
- **Первый шаг к актуальности:** реализовать вариант 2 (RAG по KB): один вызов LLM с текстами из `get_recent_news` за 1–2 дня, вывод — инсайт и sentiment; затем включить `GAME_5M_USE_LLM_NEWS=true` и подписать блок как «Инсайт LLM по новостям из KB».
- **При необходимости breaking news:** добавить вариант 3 или 4 с Web Search API и отдельным ключом в config.
