Анализатор эффективности сделок

Единый расчёт с Telegram командой /analyser. Период 1-30 дней. Ниже — человекочитаемая выжимка; полный отчёт всегда в JSON с того же API (удобно сохранить или отдать скрипту).

Расшифровка метрик и «типов потерь»

Все проценты по сделке считаются на 5m-окне от входа до выхода (Yahoo 5m, ET), плюс фактический realized_pct с учётом комиссий. Подробные формулы: файл docs/GAME_5M_CALCULATIONS_AND_REPORTING.md в корне репозитория.

Реализованный % (realized)
Фактический результат сделки в % к cost-basis (и log-return в JSON при детализации).
Missed upside (недобор)
Сколько ещё было «на столе» до максимума цены по окну: max(0, potential_best_pct − realized_pct) относительно high 5m-баров. Missed upside Σ в сводке — сумма по сделкам. Блок «По выигрышным missed Σ» — только по сделкам с realized > 0.
Avoidable loss Σ
Насколько результат хуже, чем минимум по окну: max(0, realized_pct − preventable_worst_pct) относительно low. Показывает «лишнюю просадку» относительно худшей точки, видимой на 5m внутри сделки (не прогноз).
Top losses
Сделки с наименьшим realized_pct за период. При высоком win rate сюда попадают не обязательно убытки, а самые слабые плюсы — сравнивайте с колонкой missed и exit.
Top missed upside
Крупнейший недобор (сортировка по missed), без фильтра по знаку результата: и прибыльные, и убыточные сделки.
Выигрышные с крупным недобором
Только realized > 0, упорядочены по missed: выход в плюсе при заметном запасе до high окна (ранний тейк / узкая цель).

В JSON также есть счётчики риска (например late_polling_signals, убыточные при высокой волатильности) — см. summary и trade_effects.

Результирующий JSON (локально, скрипты, jq)

Один запрос GET /api/analyzer возвращает весь отчёт: meta, summary, top_cases, llm, auto_config_override и при необходимости trade_effects. Параметры формы выше (дни, стратегия, LLM) подставляются в ссылки и в скачивание.

Открыть в браузере (можно «Сохранить как…»):

  • Полный отчёт — как после «Запустить анализ»
  • То же + массив trade_effects по каждой сделке

Путь для curl / копирования:

curl -sS "https://ВАШ_ХОСТ/api/analyzer?days=7&strategy=GAME_5M&use_llm=0" -o report.json

Сводка

Missed upside Σ — сумма недоборов до high окна; Avoidable loss Σ — сумма «лишней просадки» относительно low окна (см. раскрывающийся блок выше).

Нажмите "Запустить анализ".
