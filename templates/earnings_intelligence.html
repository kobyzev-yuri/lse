Как читать этот экран — кратко · полное руководство (RU)

Вкладки: События (календарь) · Brief · Spillover · Fusion — одинаковый экран с тулбаром; из таблицы кнопки Brief/Fusion/Spillover переключают вкладку. Тикер — поле ввода (не длинный список).

  • Регрессия (сейчас в prod): CatBoost предсказывает forward_log_ret_5d — log-доходность на 5 торговых дней после отчёта. API: /api/ml/event-reaction/{ticker}. Признаки из daily quotes + market regime; тексты earnings пока не в product feature_builder.
  • UP/DOWN/FLAT: не классификатор, а правило по порогу на фактической 5d log-return (backfill из quotes).
  • Scenario (LLM): из transcript/8-K — capex_positive_for_infra_peers, … Колонка LLM scenario в таблице.
  • Scenario (ML): CatBoost multi-class по quotes_regime_earnings_v1 — колонка ML scenario, вкладки Fusion / Shadow. Не путать с LLM.
  • Peer graph: directed edges «лидер → пир» с весом. Spillover = forward returns пиров от даты отчёта лидера (event-study), не скользящая корреляция.
  • quotes_regime_earnings_v1: backfill с peer graph + tone — run_earnings_ml_refresh.py.
  • Классификатор сценариев (pilot): train на LLM-метках; прогноз — Fusion + колонка ML; качество — вкладка Shadow.
  • Дата отчёта: колонка Date в «Событиях»; кнопки Brief/Fusion подставляют тикер и дату на своей вкладке. Telegram: /earnings META 2026-04-29.

Event Brief — один отчёт: LLM-факты, ML-слои, guidance/capex, spillover по пирам.
  • Scenario (LLM) — из extract; headline и rationale.
  • Scenario classifier (ML) — pred CatBoost + proba + ожидаемый знак source (advisory).
  • Regression (ML) — pred 5d log-return source в %.

Event Brief

Выберите строку в «Событиях» или введите тикер и дату.
Peer graph — статическая карта «кто на кого влияет после earnings», из таблицы peer_graph_edge.
  • Ребро (edge): source → target — после отчёта source (лидер, например META) ожидаем влияние на target (пир: NVDA, MU, …).
  • weight 0–1 — сила связи (чем длиннее синяя полоска в списке Edges, тем выше вес).
  • relation_type — тип связи (supplier, peer, sector и т.д., см. в списке Edges).
  • Это не дневная матрица /corr: граф задаёт кого смотреть, а фактические % — на вкладке Spillover / в Brief.

Sources (out-degree)

Тикеры-лидеры с исходящими рёбрами: сколько пиров в графе «слушают» этот source. Клик по тикеру → вкладка Spillover (история отчётов и 1d/5d пиров).

Edges

Список рёбер source → target (первые 40 из полного графа). Для полной истории движений после даты отчёта используйте Spillover или Brief.

Spillover — история отчётов source: факт 1d/5d пиров + ML pred 5d (если есть features).
  • Source — тикер-лидер; пиры из peer_graph_edge.
  • Для каждой даты: LLM scenario, ML scenario (classifier), source 5d, таблица пиров (fact + ML pred).
  • Текущий контекст (тикер + дата) — сверху отдельным блоком с полным Brief spillover.
  • Из таблицы «События» кнопка Spillover подставляет source и открывает эту вкладку.

Source = тикер из Контекст.

Shadow — таблица качества классификатора сценариев: pred ML vs факт 5d source и LLM-метка. Смотрите параллельно с колонкой ML scenario в «Событиях».
  • Pred scenario — то же, что ML scenario / Fusion (CatBoost до созревания 5d).
  • Sign — совпал ли знак 5d у source с эвристикой сценария (capex → source часто вниз, gap_up → вверх).
  • Class — pred scenario = LLM-метка llm_scenario_v0; «—» если метки нет или только UP/DOWN/FLAT.
  • Peer 5d — среднее 5d по пирам; peer sign acc в шапке — доля угадываний знака пиров.
  • Pseudo PnL — условная доходность «торговли знаком сценария» на source минус round-trip cost.
Нажмите «Загрузить shadow report».
Fusion — одно событие: регрессия 5d (source) + scenario classifier + advisory. LLM scenario — на Brief, не здесь.
  • Regression ML — «сколько» по source; Scenario ML — «какая история»; сравните с LLM на Brief.
  • Peer spillover ML — pred 5d по каждому peer (CatBoost); fact vs pred в таблице.
  • execution blocked — всегда; не подключено к GAME_5M / portfolio bot.
  • Тикер и дата — как на Brief; из таблицы кнопка Fusion подставляет их автоматически.
