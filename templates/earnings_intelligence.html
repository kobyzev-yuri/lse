Один отчёт в фокусе: календарь → обзор → ML → пиры. Системные экраны — граф, качество модели, pipeline.
Как устроен экран · полное руководство
  • Календарь — фильтр периода (по умолчанию 30д), клик по строке; самый свежий отчёт подсвечивается и выбирается автоматически.
  • Обзор / ML прогноз / Пиры / Post-mortem — четыре среза одного события; без выбора недоступны.
  • Post-mortem — после созревания 5d: pred vs fact по regression, scenario, peer spillover, fusion; расшифровки внизу вкладки.
  • Peer graph / Качество ML / ML pipeline — справочник и мониторинг; не привязаны к одному отчёту.
  • LLM scenario — из transcript/8-K; ML scenario — CatBoost classifier. Регрессия 5d — отдельный слой на вкладке ML прогноз.

Обзор — LLM-факты, guidance, цитаты, исход source и сводка по пирам для выбранного отчёта.

Выберите событие в календаре или укажите тикер и дату выше.

ML прогноз — advisory по одному событию: регрессия 5d source, classifier сценария, сводка alignment. Без исполнения сделок.

Нужен активный тикер и дата отчёта.

Пиры — forward returns пиров после даты отчёта лидера (1d/5d fact vs ML pred). Только выбранное событие.

Укажите тикер-лидер и дату его отчёта.

Post-mortem — разбор созревшего отчёта: что предсказала каждая модель и что вышло по факту 5d. Используется Trust Arbiter (T_hit).

Выберите тикер и дату отчёта в фокус-баре или календаре.

Peer graph — статическая карта «лидер → пир» из peer_graph_edge. Клик по source подставляет тикер в контекст (дату выберите в календаре).
